# Quiz: TypeScript Fundamentals

Let's check your understanding of the core concepts from this lesson.

## Which statement best describes TypeScript?

Think about the relationship between TypeScript and JavaScript, how the compiler works,
and what happens to type annotations when your code runs.

Consider these key concepts from the lesson:
- What does "superset" mean?
- What is type erasure?
- Where do types exist — at compile time or runtime?
- What does the TypeScript compiler (`tsc`) produce as output?

<div class="hint">
Remember: TypeScript *extends* JavaScript rather than replacing it. The compiler's job is to
check your types and then produce output that any JavaScript runtime can execute. After
compilation, there is no trace of TypeScript's type system in the output.
</div>
